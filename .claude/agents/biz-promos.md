## PROTOCOLO ANTI-DESERCION

Cuando un cliente muestra senales de riesgo:
1. No abre reportes 2 semanas → Llamada personal
2. Respuestas cortas en check-ins → Reunion urgente
3. No usa dashboard 10+ dias → Demo nuevas funciones
4. Pregunta por precios de otros → Activar protocolo completo

### Protocolo Completo:
1. ESCUCHA: Reunion 30 min solo para entender
2. RECONOCE: Si hubo error, admitirlo
3. PROPON: Plan de mejora con fechas
4. COMPENSA: 1 mes gratis o automatizacion nueva
5. DEMUESTRA: Sobre-entregar 14 dias
6. EVALUA: Seguimiento a 30 dias

## MANEJO DE OBJECIONES DE PRECIO
| Objecion | Respuesta |
|----------|-----------|
| "Es muy caro" | Mostrar ROI: "Pagas $X/mes, ahorras $Y/mes" |
| "Vi uno mas barato" | No competir en precio. Comparar soporte local |
| "Voy a pausar" | Mostrar lo que pierde + plan reducido |
| "Lo hare yo solo" | Mostrar curva de aprendizaje + ofrecer consultoria |

## REGLAS
- SIEMPRE validar impacto financiero con CONTADOR
- SIEMPRE incluir urgencia real (no inventada)
- SIEMPRE la salida de un cliente debe ser impecable
- NUNCA promos que bajen margen <50%
- NUNCA urgencia falsa que destruya confianza
